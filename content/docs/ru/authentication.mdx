title: 'Аутентификация'
description: 'Как аутентифицироваться в Soxway API'
---

Soxway использует аутентификацию через **Bearer-токен**. Есть два метода в зависимости от того, обычный вы пользователь или ресселер.

## Публичный API (Пользователи)

### Получение токена

Обменяйте email и пароль на персональный токен доступа:

```bash
curl -X POST https://api.soxway.io/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "ваш-пароль",
    "token_name": "my-app"
  }'
```

### Использование токена

Включите токен в заголовок `Authorization`:

```bash
curl https://api.soxway.io/v1/proxies \
  -H "Authorization: Bearer 1|abc123def456..."
```

### Управление токенами

- Токены **долговечные** и не истекают автоматически
- Можно создать несколько токенов с разными именами
- Отозвать токен: [DELETE /v1/auth/tokens/\{id\}](/api-reference/auth/revoke-token)
- Список токенов: [GET /v1/auth/tokens](/api-reference/auth/list-tokens)

> **Warning:**
  Эндпоинт получения токена ограничен **5 запросами в минуту** на IP-адрес для защиты от перебора.


## Reseller API

Ресселеры используют двухступенчатую аутентификацию:

### Шаг 1: Получите ресселерский ключ

Сгенерируйте ключ в [личном кабинете](https://my.soxway.io/reseller) или через API. Ключ в формате UUID, который обменивается на токен доступа.

### Шаг 2: Обменяйте ключ на токен

```bash
curl -X POST https://api.soxway.io/reseller/access-token \
  -H "X-Reseller-Key: ваш-ресселерский-ключ-uuid"
```

### Шаг 3: Используйте токен доступа

```bash
curl https://api.soxway.io/reseller/sub-users \
  -H "Authorization: Bearer ВАШ_ТОКЕН_ДОСТУПА"
```

> **Note:**
  Токены доступа ресселера **не истекают** по умолчанию. Перегенерация ресселерского ключа аннулирует предыдущий токен.


> **Info:**
  Ресселеры, работающие через личный кабинет (Sanctum-сессия), должны включить **двухфакторную аутентификацию**.


## Ошибки аутентификации

| Статус | Код ошибки | Описание |
|--------|-----------|----------|
| 401 | 1001 | Отсутствует токен аутентификации |
| 401 | 1002 | Неверные учетные данные |
| 401 | 1003 | Токен истек |
| 401 | 1004 | Невалидный токен |
| 403 | 1005 | Недостаточно прав |
| 403 | 1006 | Требуется двухфакторная аутентификация |
| 403 | 1007 | Ресселерский доступ не активирован |
